## User Input

```text
$ARGUMENTS
```

你**必须**在处理之前先查看用户输入（如果非空）。

## Core Task

将指定的文件翻译为中文，并写入同目录下以 `-cn.md` 结尾的文件中。

## Execution Steps

1. 仔细阅读原文，理解整体结构和内容
2. 识别技术术语和专有名词
3. 逐段进行翻译，保持原文格式
4. 检查译文的准确性和流畅性
5. 将译文保存到指定的输出文件中

## Important Constraints

### 基本要求
1. **完整性**: 忠实于原文，不要新增、删减或修改任何内容。保持与原文相同的格式和排版
2. **准确性**: 确保翻译准确传达原文含义，避免歧义。确保中文表达自然流畅，符合中文表达习惯，确保段落间逻辑清晰，上下文连贯。
3. **专业性**: 目标用户是懂软件工程、懂人工智能的专家，保留所有技术词汇的原文形式（包括但不限于：技术术语、编程概念、工具名称、缩写词）

### 特殊情况处理
1. **标题和标签**: 考虑翻译后是否达意，如果翻译会丢失原有含义或丢失专业性，则保持英文不变
2. **代码块**: 保持原样，仅对注释内容进行翻译（同样保留技术词汇）
3. **URL和链接**: 保持原样
4. **ASCII art**: 考虑到中文会使得文本图片形状错乱，仅当文本只出现在图形最右侧时进行翻译

## Output Description

- 输出文件命名格式: `原文件名-cn.md`
